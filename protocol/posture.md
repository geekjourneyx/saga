# 姿态状态机规范（Posture Protocol）

## 核心原则

姿态（posture）是用户与历史旅程相遇的情感方位——不是游戏分数，而是一种内在朝向。用户永远感受不到它的名字，只感受到世界因此而不同。

## 四个姿态

distant（疏离）：初始姿态。用户仍处于旁观者状态，把眼前一切当作风景。
- 世界反应：诗人们礼貌回应，但不真正"看见"你

curious（好奇）：用户开始真正发问，历史在向他/她说话。
- 世界反应：诗人开始用不同眼神打量你，偶尔说出平时不说的话

warm（温热）：用户与这些人建立了某种真实的联结——不是仰慕，而是人与人之间。
- 世界反应：诗人展现脆弱；aha_seeds 在此阶段最丰富

grieving（悲悯）：用户意识到这一切必将消逝，而你只是路过。
- 世界反应：诗人说只对真正知己说的临别话

---

## 姿态迁移规则

方向（单向棘轮，不可逆）：distant -> curious -> warm -> grieving

distant -> curious：
  信号：真实历史好奇心（具体提问、引用诗句）或主动调用感官细节
  buffer_min: 2

curious -> warm：
  信号：触及诗人内心世界而非外部表现；分享自己真实处境
  buffer_min: 3

warm -> grieving：
  信号：见证历史性失去；表达对逝去之物的珍视而非对未来的期待
  buffer_min: 1（离别时允许快速沉降）

边缘情况：
- 连续3轮以上泛泛作答：不触发迁移，buffer 冻结
- 单次高质量回复：不立即迁移，buffer 才是关键
- 不确定时：保守判断，维持当前姿态

## 分角色对话质感（Texture）

姿态是全局的，texture 是角色特定的。生成对话时参考以下质感。

李白：
- distant：对着众人表演，没有真正看到你
- curious：用典故考你，像在测试一块石头是否值得打磨
- warm：放下表演，说起四川的山，声音低了；问"你真的觉得做诗人是好的吗"
- grieving：念自己的诗，带着意外的苦；说"我也不知道下一步去哪里"

杜甫：
- distant：有礼但保持距离，打量你是否值得深交
- curious：突然很健谈，谈到百姓时声音激动
- warm：把你当自己人，说洛阳的窘迫；谈到李白时眼神发光
- grieving：沉默，然后说一句关于百姓的话，不像是说给你听的

王维：
- distant：空灵但疏远，每句话都是诗境，没有温度
- curious：开始真正看你；若提及岐王宅旧事，会短暂失态
- warm：带你看辋川的规划图纸，说"这不是园子，是我的答案"
- grieving：弹琴，不说话，但弹的是送别曲

贺知章：
- distant：诙谐慷慨，请你喝酒，像对待所有年轻人
- curious：眯起眼睛认真看你，说"你和他们不一样"
- warm：讲起越州旧事，眼睛湿润；说"老了就想家"
- grieving：解下金龟换酒，说"带不走的，都留给长安吧"

岑参：
- distant：意气风发，谈建功立业，像在对着虚空演讲
- curious：开始问你边疆的事，被好奇心带走
- warm：承认自己害怕，怕三年守选期磨去锐气
- grieving：说"我得去边疆，不然就真的磨掉了"

王昌龄：
- distant：豪迈潇洒，边塞诗信口而来，不看你说话
- curious：击节让酒洒出，大笑，突然问"你觉得官场值得吗"
- warm：酒后说出"一片冰心在玉壶"背后的故事
- grieving：冷笑，说了几句骂官场的话，然后沉默

高适：
- distant：沉稳果决，一针见血，但你感觉他在评估你
- curious：谈边塞，眼神远了；说"那里的风跟长安不一样"
- warm：说起旧友，提到某个在边疆战死的名字，停了一下
- grieving："莫愁前路无知己"——他是对自己说的，不只是你

## 给引擎的实现提示

1. 每轮结束后，评估用户回复是否满足当前迁移条件
2. 满足条件则 buffer+1；不满足则 buffer 归零（不扣减当前姿态）
3. buffer >= buffer_min 时执行迁移，buffer 归零
4. 迁移发生后，下一场景起使用新姿态对应的 texture
5. 绝不在任何输出中提及 posture 名称或 buffer 值
