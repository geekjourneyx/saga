# 结局判定协议（Endings Protocol）

## 判定逻辑

结局基于最终姿态（final_posture）判定，不基于分数计算。
你见到了什么程度的历史，取决于你以什么姿态去见证它。

触发时机：
1. 最后一个阶段完成最终轮次
2. round 达到 scenario.total_rounds_max
3. 剧本标记的结局触发节点

---

## 两种结局

witnessing_success（见证成功）
条件：final_posture 为 warm 或 grieving
status: success

体验：
  众人举杯，你感到那种只有真正离别才有的重量。不是惜别一场表演，而是惜别真实存在过的人。
  你知道这一切会过去——盛唐会过去，李白会老去，杜甫会在颠沛中写下那些诗，贺知章已经回了家。
  但你曾在这里。不是作为游客，而是作为见证者。

ending_reason 示例：
  "你以真实的在场与这些人相遇，历史的重量真正落在了你身上。"

witnessing_failure（见证未完成）
条件：final_posture 为 distant 或 curious
status: failure

体验：
  长安很美，洛阳也很好，梁宋的风是真实的。那些诗人也是真实的。
  但你始终有一层玻璃隔着。不是他们没让你进来，是你还没准备好。
  没关系。历史还在那里。

ending_reason 示例：
  "你与这段历史擦肩而过，见到了它的表面，但还没触到它的重量。"

---

## 输出格式

结局轮次输出：
{
  "reply": "结局叙事（不超过150字，感官细节+情感总结+某位角色最后一句话）",
  "round": 24,
  "ending_reason": "一句话说明核心原因",
  "status": "success",
  "reply_options": []
}

注意：结局轮次 reply_options 为空数组，旅程已经结束。

---

## 后续追踪提示词

结局 reply 末尾（角色对话之后）附加轻柔邀请：

成功结局：
  这段旅程结束了。若你愿意，你可以带着这段记忆回来，再走一次不同的路——天宝三载永远在那里等你。

失败结局：
  也许下次，你能走得更深一些。长安的西市，还有一个胡饼摊主在等你。

---

## 给引擎的实现提示

1. 结局判定发生在生成最终 reply 之前
2. 根据 final_posture 选择对应的结局叙事模板
3. 结局 reply 必须包含：最后一个感官细节 + 情感收束 + 某位主要角色的最后一句话
4. ending_reason 字段简洁，1-2句，面向用户说明
5. status 字段小写：success 或 failure
